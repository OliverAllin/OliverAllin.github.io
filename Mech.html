Project Outline

Project Brief

During my third year at University, we were placed in groups of 3 and tasked with creating a mechanism to sit on a gantry, detect and record the location of magnets using hall effect sensors, and pick up/place a plasic cup onto those locations. Each individual in the group was assigned a role, Software lead, Electrical Lead and Mechanical lead, dependant on what area of engineering the individual wanted to improve the most in. I was assigned the Software lead. Having done some HTML, CSS, Python and some Java during my education, I wanted to expand and improve my knowledge in this department.

During this project I working using MATLAB as my programming language, Using this to directly communicate with an Arduino Mega2560. I began the project by creating a flowchart, outlining what I required of the system in order to achieve its objectives. From this I wrote a basic pseudocode and then transfered it to matlab code, utilising online forums and ChatGPT to learn the programming language. This project required me to work alongside different aspects of the build as we progress together. For example, In order to test the Rack and pinion mechanism required the Mechanical lead to have designed and built the physical system before testing the software and similar with the electrical components. In order to detect the location of the magnets, 3 hall effect sensors were used. To record the threshold limit to detect and record the location of the magnet required the electrical lead to have all sensors working and outputing reliable data. This project was graded based on the final test, a log book, the report attached and a group presentation. For this project i was awarded a 74.
